The major features of this project are:

  • Leap Motion integration into the Source Engine.
  • Gesture panels.
  • Custom gesture recognition.
  • Interactive world space GUI panels.
  • Multiple Leap Motion devices working within the same application (see Giant Leap).
  • Positional head tracking for the Rift DK1.
  • Eletro-tactile interface integration.

The Holodeck system supports a large amount of hardware:

  • Oculus Rift (visual output).
  • 1 or more Leap Motions (world interaction).
  • A joystick that supports pitch, roll and yaw (player movement).
  • Razer Hydra (positional head tracking for the Rift DK1, not required for other Rift models).
  • Electrotactile interface (interaction output, not required but enhances VR immersion).

There are two types of users for the Holodeck system:

  1. End users simply setup all equipment required by the Holodeck system and launch a Holodeck ready game.
  2. Developers create custom gestures, place gesture panels, tactile volumes and other entities within the VR world.

Gesture Panels

Gesture panels are invisible volumes that enable gesture based control of the world. There are 3 types of gesture panels supported by Holodeck:

  1. Swipe - the user must swipe their hand in a predefined direction (or against the direction).
  2. Circle - the user must draw a circle in the air with their finger for a length of time.
  3. Tap - the user must "press a button" in a predefined direction.

Gesture Detection

The Holodeck system allows developers to create custom gestures by specifying the relative angles of the users fingers (with a degree of forgiveness).

The Holodeck system provides the ability to "listen" for each gesture type (swipe, circle, tap) regardless of the users current location. This allows level designers to perform player specific actions anywhere in the world (eg: swipe left to display the HUD, swipe right to hide). Programmers are able to extend this system by implementing dedicated gesture listeners which can detect custom gestures and use gesture information to modify the world in a more meaningful way. Lessons

  • Risk Management - Using unknown technology was risky. Using the Source Engine (a technology with which I was familiar) reduced the long term risk that another game engine would be too much to learn on top of the Rift, Leap, Hydra and Electrotactile Interface.
  • More features doesn't necessarily mean the product will be better. Sometimes it makes the product more complicated. Complete hardware setup time was usually around 10-15 minutes (more if the Giant Leap application was used).
